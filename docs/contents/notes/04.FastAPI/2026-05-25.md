## Cookie 参数模型

和查询参数模型，Cookie 和 Header 参数模型同样可以创建一个 Pydantic 模型来声明一组**参数**。以 Cookie 为例：

```python
class Cookies(BaseModel):
	...
	...
cookies: Annotated[Cookies,Cookie()]
```

回顾一下，禁止额外字段的方法：

```python
class Cookies(BaseModel):
	model_config = {"extra": "forbid"}
```


## 响应模型

函数**返回类型注解**和响应模型对比：

```python
from fastapi import FastAPI
from pydantic import BaseModel,EmailStr

app = FastAPI()

class UserOut(BaseModel):
	name: str
	email: EmailStr

class UserIn(BaseModel):
	name: str
	email: EmailStr
	password: str

# 使用 response_model
@app.get("/user1",reponse_model=UserOut)
def get_user1():
	return UserDB(name="Alice",email="...",password="...")
# 实际返回时“password”被过滤

# 使用类型注解
@app.get("/user2")
def get_user2() -> UserOut:
	return UserDB(...,password="...")
# password也会过滤
```

这两种方式，都会进行类型转换和校验，`response_model` 优先级较高，未显示声明时，类型注解默认为响应模型。

> **注**：`EmailStr` 需要安装 ：
> `pip install email-validator`


`response_model` 是路径操作装饰器参数，不是函数参数

`response_model` 可以和函数参数类型不一样，但是函数参数类型要包含 `response_model`。

```python
@app.get("/user",response_model = UserOut)
def get_user(user: UserIn) -> Any
	return user
```

但是，如果只希望用一种数据类型注解，并且能够过滤数据的话，可以用继承：

```python
class BaseUser(BaseModel):
	...
	...
class UserIn(BaseUser):
	password: str
	
def get_user(user: UserIn) -> BaseUser
	return user
```
