# 请求体

Q 1：何为请求体？

请求体是客户端发送给你的 API 的数据。
响应体是你的 API 发送给客户端的数据。

发送数据通常不使用 `GET`。常用 `POST`。

推荐使用 Pydantic 模型来声明请求体。

从 `pydantic` 中导入 `BaseModel`。

Q 2：BaseModel 是什么？

专门用来定义**数据模型**的模板类，自动做类型校验 + 自动转 JSON

**请求体代码**：

```python
from fastapi import FastAPI  
from pydantic import BaseModel  
  
class Item(BaseModel):  
    name: str  
    price: float  
    is_offer: bool = None  
  
app = FastAPI()  
  
@app.post("/items/")  
async def create_item(item: Item):  
    return item
```

此处应注意：`Pydantic` 会**自动类型转换**。比如 `price:float`，发送来的数据是 `"6.66",true,false` 都会通过，发送到后端会自动转换成 `float` 类型，除非无法转换才会报错 `422`。

若要严格指定类型对应，可以用：

```python
from pydantic import BaseModel, ConfigDict

class Item(BaseModel):  
    model_config = ConfigDict(strict = True)
```

建议正式使用接口时开启严格模式。

可以使用 FastAPI 提供的自动文档来调试

## 使用模型

可以在路径操作函数中访问并操作数据模型对象的所有属性：

```python
from fastapi import FastAPI  
from pydantic import BaseModel, ConfigDict  
  
class Item(BaseModel):  
    model_config = ConfigDict(strict = True)  
    name: str  
    price: float  
    count: int | None = None  
  
  
app = FastAPI()  
  
@app.post("/items/")  
async def create_item(item: Item):  
    item_dict = item.model_dump()  
    if item.count is not None:  
        total_price = item.price * item.count  
        item_dict.update({"total_price": total_price})  
    return item_dict
```

**补**：`item.model_dump`，将 `Pydantic` 模型对象转成 `Python dict`。

请求体可以和路径参数和查询参数混用，识别规则：

- **路径**中声明了 → 路径参数
- 参数是 python 单一类型（`int` 等）→ 查询参数
- 参数类型声明为 `Pydantic` 模型 → 请求体
