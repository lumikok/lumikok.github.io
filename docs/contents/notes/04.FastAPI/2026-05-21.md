# 请求体

## 多个参数

你可以随意地混合使用 `Path` 、`Query` 和请求体参数声明

你可以添加多个请求体参数到路径操作函数中，

比如 `item` 和 `user`：

```python
from fastapi import Body,FastAPI  
from typing import Annotated  
from pydantic import BaseModel  
  
app = FastAPI()  
  
class Item(BaseModel):  
    name: str  
    price: float  
    count: int | None = None  
  
class User(BaseModel):  
    username: str  
    full_name: str | None = None  
  
@app.put("/items/{item_id}")  
async def update_item(  
    item_id: int,  
    item: Item,  
    user: User,  
    data: Annotated[str, Body()]  
):  
    return {"item_id": item_id, "item": item, "user": user, "data": data}
```

### Body

你已经知道，参数类型为单一类型（如 `str`、`list` 等）时，默认为查询参数。

而针对请求体参数，FastAPI 提供了一个和 `Query` 和 `Path` 用法相同的 `Body` 类，用于补充声明请求体参数，同样，也可以添加额外校验和元数据参数。

### 嵌入单个请求体参数

当你只有一个请求体参数，比如 `item` 时，默认情况下，FastAPI 期望的请求体是这样的：

```json
{
	"name": "na",
	"price": 3.1,
	"count": 2
}
```

但是，如果你希望 FastAPI 期望一个和多个请求体参数时一样的 JSON 的话，可以使用一个特殊的 `Body` 参数 `embed`：

```pythpn
item: Annotated[Item, Body(embed=True)]
```

从而将原本的请求体嵌入到 `item` 键中：

```json
{
    "item": {
        "name": "na",
        "price": 3.1,
        "count": 2
    }
}
```

总结一下：

- 参数声明在路径里 → 路径参数
- 非路径参数，一般为查询参数
- 参数类型为 `Pydantic` 模型，或显式声明 `Body` → 请求体参数


## 字段

`Pydantic` 提供了一个和 `Query` 、`Path` 、`Body` 一样用法的 `Field` 类，用于在 Pydantic 模型内部，为字段添加校验或描述。

声明方式与 `Query` 那些一样，但是要注意 `default` 参数，其不能 在 Annotated 内部使用，用显式赋值形式即可（`name: str = Field(...)`）。

> 此处再次声明：`Query` 、`Path` 以及后续用到的很多其它对象，创建的都是 `Param` 类的子类的对象，而 `Param` 本身是 Pydantic 中 `FieldInfo` 的子类。
> `Field` 返回的是 `FieldInfo` 的类实例，`Body` 直接返回的也是 `FieldInfo` 的子类的对象。
> 注意：从 FastAPI 导入的那些对象本质上是返回特殊类的函数。

**总结**：上述细节了解即可，知道 FastAPI 中用到的很多类，都是互相继承的关系，用法基本一样即可。

## 嵌套模型

请求体内部类型提示也可以为复杂类型，比如 `list`、`set`。

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()


class Item(BaseModel):
    tags: set[str] = set()


@app.put("/items/{item_id}")
async def update_item(item_id: int, item: Item):
    results = {"item_id": item_id, "item": item}
    return results
```

请求体示例：

```json
{
	"tags": ["bag","rat","ball"]
}
```

**注意**：`set` 用于保存唯一数据，因此收到带有重复数据的请求时，也只会保留唯一项，且输出时也是唯一的。

在 FastAPI 中，你可以任意嵌套 Pydantic 模型，也就是将一个模型，作为类型写入另一个模型内部。

这个子模型可以是单个，也可以一组，比如 `Image` 子模型，可以写成 `list[Image]` 。

可以嵌套多层。

同样，你也可以：

```python
images: list[Item]
```

这样，JSON 请求体最外层就是一个 JSON `array`。
### 特殊的类型和校验

在请求体中，可以使用继承自 `str` 的更复杂的单一值类型，比如 `HttpUrl`，用于检验该字符串是否为有效的 URL。
