# 查询参数和字符串校验

可以为参数声明额外的校验和元数据。

使用 `Annotated` 和 `Query()` 来进行，（最推荐）

- `from typing import Annotated`
- `from fastapi import Query`

通用的校验和元数据：

- `alias`：起别名，放在 Query()里面
- `title`：参数的“标题”，替代变量名，仅显示在 OpenAPI 文档中，帮助 API 使用者理解含义，写在 Query()里面
- `description`：描述信息，同上
- `deprecated = True`：标记参数弃用，同上，有别人还在用，告知弃用
- `include_in_schema = False`：从 OpenAPI 中排除参数（同样也不会展示在文档里）。

字符串特有：

- `min_length`：长度最小值
- `max_length`：同上，控制长度
- `pattern`：正则表达式（需补 python）


任何类型的默认值（包括 `None`）都会让该参数变为可选（非必填）。

要声明类型为 `list` 的查询参数（如上例），你需要显式地使用 `Query`，否则它会被解释为请求体。

```python
async def read_items(q: Annotated[list[str] | None, Query()] = None):
```

同样，你也可以把默认值设置为列表：`["foo","bar"]`。

当然，你也可以只使用 `list`，而不是 `list[str]`。


## 自定义校验

校验要求个性化时，可以使用**自定义校验函数**，但顺序在正常校验之后。

在 `Annotated` 中使用 [Pydantic 的 `AfterValidator`](https://docs.pydantic.dev/latest/concepts/validators/#field-after-validator) 来实现。

```python

def check_value(id: str):
	if not id.startswith(("a-","b-")):
```

# 路径参数和数值校验

与查询参数类似，使用 `Path` 来添加。

`from fastapi import Path`

路径参数永为必需参数

如果你将带有“默认值”的参数放在没有“默认值”的参数之前，Python 会报错。

FastAPI 也必须遵守，但是可以用 Annotated 来解决，因为没传默认值。

也可以第一个参数传 `*`，将后面的参数都变成关键字参数（键值对），不再受“默认值必须在后”的限制。

## 数值校验

- 大于等于：ge=...
- 大于：gt=...
- 小于等于：le=...
- 小于：lt=...
- float 类型也适用


`Query`、`Path` 以及你后面会看到的其他类，都是一个通用 `Param` 类的子类。

它们都共享相同的参数，用于你已看到的额外校验和元数据。

当你从 `fastapi` 导入 `Query`、`Path` 和其他对象时，它们实际上是函数。

当被调用时，它们会返回同名类的实例。

也就是说，你导入的是函数 `Query`。当你调用它时，它会返回一个同名的 `Query` 类的实例。

# 查询参数模型

当你的一部分查询参数具有相关性时，可以创建一个 `Pydantic` 模型来声明它们

这样的话，就能多个地方复用模型，并一次性声明验证以及元数据

```python
class FilterParams(BaseModel):
	limit: int = Field(100, gt=0, le=100)
	offset: int = Field(0, ge=0)
	tags: list[str] = []

async def read_items(filter_query: Annotated[FilterParams, Query()]):
```

**FastAPI** 将会从请求的**查询参数**中**提取**出**每个字段**的数据，并将其提供给你定义的 Pydantic 模型。

可以限制你要接收的参数

你可以使用 Pydantic 的模型配置来 `forbid` 任何 `extra` 字段

意思是仅接收模型内定义的参数

```python
class FilterParams(BaseModel):
	model_config = {"extra": "forbid"}
```
