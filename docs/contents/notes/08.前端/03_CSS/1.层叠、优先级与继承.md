# 层叠、优先级与继承

CSS 中的基本概念——层叠（Cascading）、优先级（Specificity）与继承（Inheritance）是理解和使用 CSS 的关键。掌握这些概念有助于我们更好地控制样式的应用和冲突解决。

## 冲突规则

当多个 CSS 规则应用于同一个元素时，浏览器需要决定使用哪个规则。这时，层叠、优先级和继承就发挥了作用。

### 层叠（Cascading）

层叠。简单来说就是 CSS 规则的顺序。后定义的规则会覆盖先定义的规则。例如：

```css
p {
  color: blue;
}
p {
  color: red;
}
```

在这个例子中，段落文本最终会显示为红色，因为后面的规则覆盖了前面的规则。

### 优先级（Specificity）

优先级是来决定当多个规则有不同选择器对应相同元素时，哪个规则会被应用。优先级的计算基于选择器的类型和数量。一般来说，优先级的计算规则如下：

1. 内联样式（如`style="..."`）的优先级最高。
2. ID 选择器（如`#id`）的优先级高于类选择器（如`.class`）、属性选择器（如`[type="text"]`）和伪类选择器（如`:hover`）。
3. 类选择器、属性选择器和伪类选择器的优先级高于元素选择器（如`div`、`p`）和伪元素选择器（如`::before`）。
4. 通配符选择器（`*`）的优先级最低。

### 继承（Inheritance）

继承是指某些 CSS 属性会从父元素传递给子元素。例如，文本颜色（`color`）和字体（`font-family`）通常会被子元素继承，而`width`和`margin`等属性则不会被继承。

```css
body {
  color: green;
}
p {
  font-size: 16px;
}
```

在这个例子中，段落`<p>`元素会继承`body`元素的绿色文本颜色。

## 协同工作

层叠、优先级和继承共同决定了最终应用于元素的样式。当多个规则冲突时，浏览器会按照以下顺序来决定使用哪个规则：

1. 首先考虑优先级，优先级高的规则胜出。
2. 如果优先级相同，则考虑规则的顺序，后定义的规则胜出。
3. 继承的属性会在没有明确指定样式时生效。

## 继承详解

比如：

```html
<ul class="main">
  <li>Item 1</li>
  <li>
    Item 2
    <ul>
      <li>2.1</li>
      <li>2.2</li>
    </ul>
  </li>
  <li>
    Item 3
    <ul class="special">
      <li>
        3.1
        <ul>
          <li>3.1.1</li>
          <li>3.1.2</li>
        </ul>
      </li>
      <li>3.2</li>
    </ul>
  </li>
</ul>
```

```css
.main {
  color: rebeccapurple;
  border: 2px solid #ccc;
  padding: 1em;
}

.special {
  color: black;
  font-weight: bold;
}
```

在这个例子中，`.main` 类定义了文本颜色为紫色，而 `.special` 类定义了文本颜色为黑色并加粗。由于继承的关系：

- `Item 1`、`Item 2` 和 `Item 3` 会显示为紫色。
- `2.1` 和 `2.2` 也会显示为紫色，因为它们继承自 `Item 2`。
- `3.1` 和 `3.2` 会显示为黑色并加粗，因为它们继承自 `.special` 类。
- `3.1.1` 和 `3.1.2` 会显示为黑色并加粗，因为它们继承自 `3.1`。

### 控制继承

CSS 提供了一些属性来控制继承行为：

- `inherit`：允许子元素继承父元素的属性值。
- `initial`：将属性值重置为其初始值。
- `unset`：如果属性是继承属性，则继承父元素的值；否则将其重置为初始值。
- `revert`：将应用于选定元素的属性值重置为浏览器的默认样式。通常与`unset`类似。
- `revert-layer`：将应用于选定元素的属性值重置为在上一个样式层中定义的值。适用于多层样式表的情况。

示例：

```html
<ul>
  <li>默认<a href="#">链接</a>颜色</li>
  <li class="class1">继承<a href="#">链接</a>颜色</li>
  <li class="class2">重置<a href="#">链接</a>颜色</li>
  <li class="class3">取消继承<a href="#">链接</a>颜色</li>
</ul>
```

```css
body {
  color: green;
}

.class1 a {
  color: inherit; /* 继承父元素颜色 */
}

.class2 a {
  color: initial; /* 重置为浏览器默认颜色 */
}

.class3 a {
  color: unset; /* 取消继承，重置为初始值 */
}
```

在这个例子中：

- `默认链接颜色` 会显示为浏览器默认的链接颜色（通常是蓝色）。
- `继承链接颜色` 会显示为绿色，因为它继承了父元素的颜色。
- `重置链接颜色` 会显示为浏览器默认的链接颜色。
- `取消继承链接颜色` 对于`<a>`元素来说，`color`属性是可继承的，因此它会显示为绿色，因为它继承了父元素的颜色。

### 重设所有属性值

有时我们希望重设一个元素的所有属性值，可以使用`all`属性：

```html
<blockquote>
  <p>当前块引用设置了样式</p>
</blockquote>

<blockquote class="fix-this">
  <p>当前块引用未设置样式</p>
</blockquote>
```

```css
blockquote {
  border: 2px solid #ccc;
  padding: 1em;
  background-color: lightyellow;
  color: brown;
}
.fix-this {
  all: unset;
}
```

在这个例子中，`.fix-this` 类使用了 `all: unset;`，这会重置该块引用元素的所有样式属性，使其不再继承或应用任何样式。

## 层叠详解

层叠是 CSS 的核心概念之一，它决定了当多个样式规则应用于同一个元素时，哪个规则最终会生效。层叠的规则重要性由低到高依次为：

1. **资源顺序**
2. **优先级**
3. **重要程度**

### 资源顺序

就是后定义的规则会覆盖先定义的规则。

### 优先级

上面已经介绍了优先级的内容。但需要注意的是，并不会覆盖所有规则，只覆盖相同的属性。

因此，惯常的做法是为基本元素定义通用样式，然后为特定元素定义更具体的样式。如：

```html
<h2>未设置类的标题</h2>
<h2 class="small">设置了 small 类的标题</h2>
<h2 class="bright">设置了 bright 类的标题</h2>
```

```css
h2 {
  color: black;
  font-size: 2em;
}
.small {
  font-size: 1.5em;
}
.bright {
  color: orange;
}
```

在这个例子中：

- 第一个`<h2>`元素会显示为黑色，字体大小为 2em。
- 第二个`<h2 class="small">`元素会显示为黑色，字体大小为 1.5em。
- 第三个`<h2 class="bright">`元素会显示为橙色，字体大小为 2em。

那么，浏览器是如何计算优先级的呢？本质上，不同类型的选择器有不同的分值，具体如下：

一个选择器的优先级可以表示为一个四位数 (a,b,c,d)：

- a：内联样式的数量（0 或 1）
- b：ID 选择器的数量
- c：类选择器、属性选择器和伪类选择器的数量
- d：元素选择器和伪元素选择器的数量

**注意**：通配符选择器（`*`）、组合符（如`+`、`>`、`~`）、调整优先级的选择器（如`:where()`）不影响优先级计算。虽然否定（`:not()`）和任意匹配`:is()`本身不影响优先级，但其参数会影响优先级计算。参数中，对优先级贡献最高的那个会被计入总优先级。

示例如下：

```html
<div class="container" id="main">
  <div class="container" id="secondary">
    <ul>
      <li class="nav"><a href="#">示例一</a></li>
      <li class="nav"><a href="#">示例二</a></li>
    </ul>
  </div>
</div>
```

```css
/* 1.优先级：0-1-0-1 */
#main a {
  background-color: lightblue;
}

/* 2.优先级：0-2-0-1 */
#secondary #main a {
  background-color: lightgreen;
}

/* 3.优先级：0-1-0-4 */
#main div ul li a {
  color: red;
}

/* 4.优先级：0-1-1-3 */
#main div ul .nav a {
  color: orange;
}

/* 5.优先级：0-0-2-4 */
div div li:nth-child(2) a:hover {
  border: 2px solid purple;
}

/* 6.优先级：0-0-3-3 */
div div .nav:nth-child(2) a:hover {
  border: 2px solid green;
}

/* 7.优先级：0-0-2-3 */
div li:nth-child(2) a:hover {
  border: 2px solid black;
}
a {
  display: inline-block;
  padding: 0.5em 1em;
  text-decoration: none;
  margin: 0.5em;
  border: 2px solid transparent;
  width: 100px;
  margin-bottom: 1em;
  text-align: center;
  line-height: 1.5;
}

ul {
  padding: 0;
}

li {
  list-style-type: none;
}
```

在这个例子中，不同的选择器具有不同的优先级，浏览器会根据优先级来决定应用哪个样式规则。

### 内联样式

内联样式，即`style`属性定义的样式，优先级最高。

### !important

有一个较为特殊的规则是`!important`，它可以提升某个样式规则的优先级，使其覆盖其他规则。使用时需要谨慎，因为过度使用`!important`会导致样式难以维护。

## CSS 位置的重要性

CSS 声明的优先级取决于定义它的样式表和级联层。

它可以让用户设置自定义样式表来覆盖作者的样式表，也可以在级联层中声明开发者样式表。

### 覆盖声明的顺序

后一种声明会覆盖前一种声明：

1. 用户代理样式表中的声明（如浏览器默认样式）
2. 用户样式表中的常规声明（用户自定义样式）
3. 作者样式表中的常规声明（网页设计者定义的样式）
4. 作者样式表中的`!important`声明
5. 用户样式表中的`!important`声明
6. 用户代理样式表中的`!important`声明

### 级联层

级联层是比较高级的概念，虽然有时并不会直接用到，但仍然需要去了解层是如何级联的。
