# 构建文档



HTML 文档是由一系列嵌套的元素构成的，这些元素定义了文档的结构和内容。

## 基本组成部分


- **页眉**（Header）：通常包含网站的标题、导航菜单等信息，位于页面的顶部。通常在各个页面中保持一致。
- **导航栏**（Navigation Bar）：提供网站各个部分的链接，帮助用户快速找到所需内容。
- **主内容区**（Main Content Area）：这是页面的核心部分，包含主要的信息和内容。
- **侧边栏**（Sidebar）：通常包含辅助信息、链接或广告，位于主内容区的侧面。
- **页脚**（Footer）：位于页面的底部，通常包含版权信息、联系方式和其他相关链接。
- **文章**（Article）：表示独立的内容块，适用于博客文章、新闻报道等。

## 语义化标签



HTML5 引入了一些新的语义化标签，使得文档结构更加清晰，下面介绍对应上面组成部分的标签：

- `<header>`：定义页眉部分。
- `<nav>`：定义导航栏。
- `<main>`：定义主内容区。主内容中还可以使用 `<section>`（章节）和 `<article>`（文章）以及 `<div>`（通用容器）等标签来组织内容。
- `<aside>`：定义侧边栏内容。通常放在`<main>`中。
- `<footer>`：定义页脚部分。
- `<section>`：定义文档中的章节，通常包含一个标题。
- `<article>`：定义独立的内容块，适用于博客文章、新闻报道等。

## 示例结构



```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>示例文档结构</title>
  </head>
  <body>
    <header>
      <h1>网站标题</h1>
      <nav>
        <ul>
          <li><a href="#">首页</a></li>
          <li><a href="#">关于我们</a></li>
          <li><a href="#">服务</a></li>
          <li><a href="#">联系</a></li>
        </ul>
      </nav>
    </header>

    <main>
      <article>
        <h2>文章标题</h2>
        <p>这是文章的主要内容。</p>
      </article>

      <aside>
        <h3>侧边栏标题</h3>
        <p>这里是一些辅助信息。</p>
      </aside>
    </main>

    <footer>
      <p>© 2024 公司名称. 保留所有权利。</p>
    </footer>
  </body>
</html>
```

## 元素细节



当然，理解所有这些标签的细节和用法是很重要的。但以下是主要元素的简要说明：

- `<main>`：每个 HTML 文档只能有一个 `<main>` 元素，它应该包含页面的主要内容，且直接位于 `<body>` 内。
- `<article>`：包围的内容即一篇文章，与页面的其他内容相对独立，可以单独分发或重用。
- `<section>`：用于将内容分成不同的章节，每个章节通常包含一个标题。
- `<aside>`：用于包含与主内容相关但不直接属于主内容的信息，如侧边栏、广告等。
- `<header>` ：是简介形式的内容。如果一个 `<header>` 元素位于 `<body>` 内，它通常包含整个页面的介绍信息；如果它位于 `<article>` 或 `<section>` 内，它则包含该部分的介绍信息。
- `<footer>`：通常包含版权信息、联系方式和其他相关链接，可以位于 `<body>` 内或其他元素（如 `<article>` 或 `<section>`）内。
- `<nav>`：用于定义导航链接的部分，帮助用户在网站中浏览不同的页面或部分。

### 无语义包装元素


有时我们需要一个没有特定语义的容器来组织内容，这时可以使用 `<div>` 和 `<span>`元素，并配合使用 `class` 属性提供一些标签，使这些元素能易于定位和样式化。

`<span>`是一个行级无语义元素，最好只用于无法使用其他语义标签的少量文本包装的场景。

`<div>`是一个块级无语义元素，适用于需要分组较大内容块的场景。

**注意**：虽然 `<div>` 和 `<span>` 可以用于布局和样式化，但应尽量使用语义化标签来提高文档的可读性和可维护性。

### 换行与水平分割线



- `<br>`：用于在文本中插入换行符，使内容在显示时另起一行。它是一个自闭合标签，不需要结束标签。
- `<hr>`：用于在内容中插入水平分割线，表示主题的转换或内容的分隔。它也是一个自闭合标签，不需要结束标签。
