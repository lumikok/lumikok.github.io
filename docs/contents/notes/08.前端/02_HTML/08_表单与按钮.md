# 表单与按钮


当需要在网页中实现双向交互时，就需要用到表单与按钮元素。表单用于收集用户输入的数据，而按钮则用于触发某些操作，如提交表单或执行脚本。

## 简述


按钮通常是通过 `<button>` 元素创建的（也可以使用 `<input type="button">`）。这些按钮可以与任意的脚本语言（如 JavaScript）结合使用，以实现动态交互效果。

表单则是通过 `<form>` 、`<label>` 、`<input>` 和 `<select>` 等元素创建的。与按钮相比，表单更复杂，因为它们包含多个输入字段和控件，用于收集各种类型的数据。

## 按钮



按钮用于触发某些操作。最常见的按钮类型是 `<button>` 元素。它可以包含文本、图像或其他 HTML 元素，基本语法如下：

```html
<button>点击我</button>
```

下面具体介绍其详细属性：

### 核心属性



```html
<!-- type 属性 - 定义按钮的行为 -->
<button type="button">普通按钮</button>
<button type="submit">提交按钮</button>
<button type="reset">重置按钮</button>

<!-- disabled 属性 - 禁用按钮 -->
<button disabled>禁用按钮</button>

<!-- name和value 属性 - 定义按钮的名称和值 -->
<button name="myButton" value="value1">按钮</button>

<!-- autofocus 属性 - 页面加载时自动聚焦按钮 -->
<button autofocus>自动聚焦按钮</button>
```

具体属性说明：

- `type`：定义按钮的行为。可选值有 `button`（普通按钮）、`submit`（提交表单）和 `reset`（重置表单）。区别如下：
    - `button`：普通按钮，点击时不会执行任何默认操作，通常与脚本结合使用。
    - `submit`：提交按钮，点击时会提交所属表单的数据到服务器。
    - `reset`：重置按钮，点击时会将所属表单的所有输入字段恢复到初始值。
- `disabled`：禁用按钮，使其不可点击。
- `name` 和 `value`：定义按钮的名称和值，通常用于表单提交时传递数据。
- `autofocus`：页面加载时自动聚焦按钮。

### 表单属性



按钮还可以与表单元素结合使用，以下是一些常见的表单属性：

```html
<form id="myForm">
  <!-- 表单内容 -->
</form>

<!-- form 属性 - 指定按钮所属的表单 -->
<button form="myForm" type="submit">提交表单</button>

<!-- 覆盖 form 属性 - 覆盖按钮的默认行为 -->
<button
  type="submit"
  formaction="submit_page.html"
  formmethod="post"
  formenctype="multipart/form-data"
  formnovalidate
  formtarget="_blank"
>
  自定义提交按钮
</button>
```

具体属性说明：

- `form`：指定按钮所属的表单，通过表单的 `id` 属性值进行关联。
- `formaction`：覆盖按钮的默认提交 URL，指定表单提交时的目标地址。
- `formmethod`：覆盖按钮的默认提交方法，指定表单提交时使用的 HTTP 方法（如 `get` 或 `post`），区别如下：
    - `get`：将表单数据附加在 URL 后面，适用于非敏感数据的提交。
    - `post`：将表单数据包含在请求体中，适用于敏感数据或大量数据的提交。
- `formenctype`：覆盖按钮的默认编码类型，指定表单提交时的数据编码方式（如 `application/x-www-form-urlencoded`、`multipart/form-data` 等）。
- `formnovalidate`：覆盖按钮的默认验证行为，指定提交表单时不进行验证。
- `formtarget`：覆盖按钮的默认目标，指定表单提交时打开结果的窗口或标签页（如 `_self`、`_blank`、`_parent`、`_top`）。

**补充**：按钮通常与 JavaScript 结合使用，以实现更复杂的交互效果。例如，可以通过事件监听器来响应按钮点击事件，从而执行特定的操作。

## 表单



### 基本结构


一个基础的表单包含以下三个主要部分：

- `<form>` 元素：包裹表单的所有内容，所有包含在 `<form>` 标签内的表单控件都会被视为该表单的一部分，它们的数据会在提交时一起发送到服务器。
- 一个或多个 `<label>` 元素与表单空间元素：
    - 表单控件元素允许用户选择或输入数据，这些数据会随着表单一起提交到服务器。常见的表单控件元素包括：
        - `<input>`：用于各种类型的输入，如文本、密码、复选框、单选按钮等。
        - `<select>`：用于创建下拉列表。
        - `<textarea>`：用于多行文本输入。
    - `<label>` 元素：为表单控件提供标识标签，用于描述应在控件中输入或选择的数据。
- `<button>` 元素：用于提交表单或触发其他操作。

示例如下：

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>表单示例</title>
  </head>
  <body>
    <form action="/submit" method="post">
      <label for="username">用户名:</label>
      <input type="text" id="username" name="username" required />

      <label for="password">密码:</label>
      <input type="password" id="password" name="password" required />

      <button type="submit">提交</button>
    </form>
  </body>
</html>
```

当然，仅有这些元素，还不足以实现提交功能，下面将介绍表单部分的详细属性。

### `<form>` 元素



`<form>` 元素用于创建一个表单，将所有表单控件组织到一起。上述示例中仅使用了两个最重要的属性：

- `action`：指定表单提交的目标 URL，用于指定提交的表单数据将发送到哪里进行处理。
- `method`：指定将表单数据发送到服务器时使用的数据传输方法，常见的值有 `get` 和 `post`，当前不用过分纠结两者的区别。

#### 检查提交的数据



在实际开发中，表单提交后，服务器端会接收到用户输入的数据。为了检查提交的数据，可以使用浏览器的开发者工具（通常按 F12 打开），在“网络”或“Network”选项卡中查看表单提交的请求。通过查看请求的详细信息，可以看到提交的数据内容。

#### 结构化表单



我们可以在`<form>` 元素中包含任意的 HTML 元素，用于组织表单控件本身，并且可以位 CSS 样式提供可定位的容器。比如用`<p>`标签来分割表单控件，也可以用`<div>`、`<section>`甚至是`<li>`标签来组织表单控件。除此之外，还有一些专门用于表单布局的标签，比如`<fieldset>`和`<legend>`标签，当表单较为复杂时，可以使用这些标签来分组相关的表单控件，从而提高表单的可读性和可用性。

### `<input>` 元素



`<input>` 元素用于创建各种类型的输入控件。它是一个自闭合标签，如下所示：

```html
<input type="text" name="username" id="username" required />
```

具体解释：

- `type`：定义输入控件的类型，如 `text`（文本输入）、`password`（密码输入）、`checkbox`（复选框）、`radio`（单选按钮）等。
- `name`：定义输入控件的名称，用于标识提交的数据。
- `id`：定义输入控件的唯一标识符，通常与 `<label>` 元素的 `for` 属性关联。
- `required`：指定该输入控件为必填项，用户必须填写才能提交表单。

**注意**：有些输入类型的值并不是用户直接输入的，而是通过选择或其他方式获得的。例如，`checkbox` 和 `radio` 类型的输入控件允许用户选择预定义的选项，而不是手动输入文本。

对于单选按钮和复选框，一般需要在`<value>`属性中指定具体的值，以便在表单提交时传递正确的数据。

#### `reqquired`和`value`属性的实际效果



1. 当有`required`属性时，如果用户没有填写该输入框，浏览器会阻止表单提交，并提示用户填写该字段。
2. 如果在`<input>`元素中设置了`value`属性，那么该输入框在页面加载时会显示该值作为默认值。如果用户没有修改该值，提交表单时会将该默认值发送到服务器。
3. 对于复选框和单选按钮，如果设置了`value`属性，当用户选择该选项时，提交表单时会将该值发送到服务器。如果没有设置`value`属性，默认情况下，浏览器会发送`on`作为值。

#### 特定类型的文本输入框



当我们创建文本输入框时，可以通过设置不同的 `type` 属性来指定输入框的类型。并且，不同类型的输入框会有不同的行为和验证规则。例如`email`类型的输入框会验证用户输入的内容是否符合电子邮件格式。

### `<label>` 元素



正如上文所述，`<label>` 元素用于为表单控件提供标识标签，用于描述应该在控件中输入或选择的数据。标签与空间之间的关联方式是：给表单控件设置一个唯一的 `id` 属性值，然后在 `<label>` 元素中使用 `for` 属性引用该 `id`。例如：

```html
<label for="name">姓名:</label> <input type="text" id="name" name="name" />
```

`<label>`元素的好处有很多，以下几点尤为重要：

- **无障碍性**：使用 `<label>` 元素可以提高表单的无障碍性，帮助使用屏幕阅读器的用户更好地理解表单内容。
- **点击区域扩大**：点击 `<label>` 元素时，会自动聚焦到关联的表单控件上，提升用户体验。

#### 显式与隐式关联



上述示例中展示的是显式关联，即通过 `for` 属性将 `<label>` 与特定的表单控件关联起来。除此之外，还有一种隐式关联方式，即将表单控件嵌套在 `<label>` 元素内，例如：

```html
<label>
  姓名:
  <input type="text" name="name" />
</label>
```

这种方式同样可以实现标签与控件的关联，且不需要使用 `for` 属性，但在复杂表单中，显式关联通常更为清晰和易于维护。

### `<button>` 元素



当一个`button`元素放在`form`表单内时，默认类型是`submit`，即点击该按钮会提交表单。如果希望创建一个普通按钮，可以将其`type`属性设置为`button`（然后通过 JavaScript 来定义按钮的行为）。

## 无障碍性



在创建表单和按钮时，考虑无障碍性是非常重要的。以下是一些建议：

- 使用 `<label>` 元素为每个表单控件提供描述标签。
- 确保按钮具有明确的文本或图标，以便用户理解其功能
- 使用适当的对比度和颜色，以确保按钮和表单控件在各种设备上都易于识别和使用。
- 尽量使用语义化的 HTML 元素，以提高屏幕阅读器的兼容性。

## 其他控件类型



下面是一个稍复杂的表单示例，包含了多种表单控件类型：

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>复杂表单示例</title>
  </head>
  <body>
    <form action="./payment_page" method="get">
      <h2>报名表单</h2>
      <fieldset>
        <legend>选择课程：</legend>
        <div>
          <input
            type="radio"
            id="course1"
            name="course"
            value="economy"
            checked
          />
          <label for="course1">经济学入门</label>

          <input type="radio" id="course2" name="course" value="history" />
          <label for="course2">世界历史概览</label>

          <input type="radio" id="course3" name="course" value="programming" />
          <label for="course3">编程基础</label>
        </div>
      </fieldset>
      <fieldset>
        <legend>选择附加服务：</legend>
        <div>
          <input
            type="checkbox"
            id="service1"
            name="service"
            value="tutoring"
          />
          <label for="service1">一对一辅导</label>
          <input
            type="checkbox"
            id="service2"
            name="service"
            value="materials"
          />
          <label for="service2">学习资料包</label>
        </div>
      </fieldset>
      <p>
        <label for="paymentMethod">选择支付方式：</label>
        <select id="paymentMethod" name="paymentMethod">
          <option value="">请选择</option>
          <option value="creditCard">信用卡</option>
          <option value="paypal">PayPal</option>
          <option value="bankTransfer">银行转账</option>
        </select>
      </p>
      <p>
        <label for="comments">备注：</label>
        <textarea
          id="comments"
          name="comments"
          rows="4"
          cols="50"
          placeholder="请输入您的备注信息..."
        ></textarea>
      </p>
      <p>
        <button type="submit">提交报名</button>
      </p>
    </form>
  </body>
</html>
```

详解：

- 使用 `<fieldset>` 和 `<legend>` 元素对相关的表单控件进行分组，提高表单的可读性。
- 使用单选按钮（`<input type="radio">`）让用户选择课程。
- 使用复选框（`<input type="checkbox">`）让用户选择附加服务。
- 使用下拉列表（`<select>`）让用户选择支付方式。
- 使用多行文本框（`<textarea>`）让用户输入备注信息。
- 使用提交按钮（`<button type="submit">`）提交表单。
- 每个表单控件都有对应的 `<label>` 元素，确保无障碍性。

### 单选按钮



上述“选择课程”部分使用了单选按钮（`<input type="radio">`），用户只能选择其中一个课程。这种控件适用于互斥选项的场景。

`radio` 输入控件的工作方式如下：

- 所有具有相同 `name` 属性值的单选按钮被视为一组，用户在该组中只能选择一个选项。
- 每个单选按钮都有一个唯一的 `id` 属性，用于与对应的 `<label>` 元素关联。
- 每个单选按钮都有一个`value` 属性，用于指定提交时该按钮对应的值。提交的数据是一个键值对，键是 `name` 属性的值，值是所选单选按钮的 `value` 属性的值。
- 每个单选按钮的`<label>`应该描述该选项的含义，而非该组。而对整组单选按钮的描述，应该使用`<fieldset>`和`<legend>`标签来实现，如上例所示。

**补充**：`checked`属性用于指定默认选中的单选按钮。在上述示例中，`course1` 被设置为默认选中项。这意味着你必须选择一个选项才能提交表单。

#### 禁用表单控件



有时需要禁用某些表单控件，使其不可用。可以使用 `disabled` 属性来实现这一点。这样，用户将无法与该控件进行交互。

### 复选框



"选择附加服务" 部分使用了复选框（`<input type="checkbox">`），用户可以选择多个选项。复选框适用于非互斥选项的场景。

其工作方式与单选按钮类似，主要区别在于：每个复选框的`name`属性是不同的，且通常不会设置`value`属性。这意味着，每个复选框代表不同的数据值，用户可以选择多个选项，提交的数据将包含所有被选中的复选框的值。

**注意**：给复选框添加`value`属性意义不大，其提交给服务器的具体值并不重要，重要的是该复选框是否被选中。

### 下拉列表



下拉列表使用 `<select>` 元素创建，允许用户从预定义的选项中进行选择。每个选项由 `<option>` 元素表示。每个 `<option>` 元素都有一个 `value` 属性，用于指定提交时该选项对应的值，若未指定，则默认使用选项的文本内容作为值。

**补充**：可以使用 `selected` 属性来指定默认选中的选项。

### 多行文本框



多行文本框使用 `<textarea>` 元素创建，允许用户输入多行文本。与 `<input>` 元素不同，`<textarea>` 不是自闭合标签，需要有一个结束标签。可以使用 `rows` 和 `cols` 属性来指定文本框的行数和列数。

大多数浏览器还会在文本框中显示一个可调整大小的控件，允许用户通过拖动来调整文本框的大小。

## 表单验证



表单验证主要分为两种类型：客户端验证和服务器端验证。

- **客户端验证**：在用户提交表单之前，由浏览器执行的验证。可以使用 HTML5 提供的内置验证属性（如 `required`、`pattern`、`min`、`max` 等）来实现基本的验证需求。客户端验证可以提高用户体验，减少无效数据提交到服务器的可能性。
- **服务器端验证**：在服务器接收到表单数据后执行的验证。服务器端验证是确保数据安全和完整性的最后一道防线，必须始终进行。即使已经进行了客户端验证，也不能完全依赖它，因为用户可能会绕过客户端验证。
