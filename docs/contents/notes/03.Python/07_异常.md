## 介绍

- **异常**：也叫 **`Bug`**，就是程序运行过程中出现的错误，会中断程序的正常运行流程
- **作用**：
	- 保证程序中数据、逻辑的正确性，避免程序执行混乱
	- 在开发阶段，尽量多发现问题，尽早解决问题

## 常见异常

**python常见异常如下**：

1. **`NameError` 名字错误**
    
    - 报错原因：调用了**从未定义、未声明**的变量 / 函数名，Python 解释器在当前作用域找不到该标识符。
    - 典型场景：直接打印未赋值的变量、调用未定义的函数。
    
2. **`TypeError` 类型错误**
    
    - 报错原因：**数据类型使用错误**，对不兼容的类型执行操作，函数传入了不符合要求的参数类型。
    - 典型场景：字符串与整数直接相加、函数参数类型传错。
    
3. **`IndexError` 索引越界错误**
    
    - 报错原因：**有序序列（列表 list、元组 tuple、字符串 str）下标超出有效范围**。Python 序列索引从 `0` 开始计数。
    - 典型场景：列表只有 3 个元素，强行访问下标 `5`、`10` 等不存在的位置。
    
4. **`KeyError` 键错误**
    
    - 报错原因：**字典（dict）中访问了不存在的键（key）**。
    - 典型场景：字典里没有`"age"`这个标签，却执行 `dict["age"]` 取值操作。
    
5. **`ValueError` 值错误**
    
    - 报错原因：数据**类型本身合法，但数值 / 内容不符合操作要求**，无法完成对应转换与运算。
    - 典型场景：把字母字符串 `int("abc")` 强行转换为整数。

## 异常处理

**异常处理方式如下**：

1. **不处理**：整个程序会因为一个 `Bug`，中断执行
2. **捕获处理**：按照我们自己的处理方式，处理完异常，程序继续执行（也就是**写好预案**）

**具体代码如下**：

```python
try:
	可能出现异常的代码块

except 异常类型 as 变量名:
	出现异常时的预案

finally:
	不管是否出现异常，都会执行的代码 
```

**注意**：
`except` 语句可以写多个

### 异常传递

- 异常传递就是异常在函数调用过程中层层上报的过程，直到有人处理它，或者程序崩溃

**异常处理代码样例如下**：

```python
while True: 
			try: 
	            print("========== 教务成绩管理系统 ==========")  
	            print("1. 添加学生成绩")  
	            print("2. 修改学生成绩")  
	            print("3. 删除学生成绩")  
	            print("4. 查询指定学生成绩")  
	            print("5. 展示全部学生成绩")  
	            print("0. 退出系统")  
	  
	            choice = int(input("请输入选择："))  
	  
	            match choice:  
	                case 1:  
	                    self.add_student()  
	                case 2:  
	                    self.update_score()  
	                case 3:  
	                    self.delete_student()  
	                case 4:  
	                    self.find_student()  
	                case 5:  
	                    self.show_all()  
	                case 0:  
	                    print("系统成功退出！")  
	                    break  
	                case _:  
	                    print("错误！请重新选择 \n")
	        except ValueError as e:
		        print("输入的内容不合法！！！请重新输入")
		    except Exception as e:
			    print("程序运行出错了，请重新选择")    
```

**注意**：
- **`Exception`**：是通用错误捕获，必须放在错误处理的最后捕获
