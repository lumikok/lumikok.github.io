# 类型提示

目录：

## 前言

现代简洁写法的类型提示语法是 `python3.10+` 添加的语言特性。

类型提示用于声明变量的数据类型，声明方式为：`变量: 数据类型`。

**需谨记**：类型提示用于给编辑器和相关工具提示信息，使得编辑器代码补全提示更强，且能够自动检测类型并给出提示信息，便利开发工作，**但是本质不影响代码运行，仅提示，非约束**。

下面简单介绍了一部分类型提示的语法。

## 声明类型

日常开发过程中，声明类型提示的主要应用场景是函数参数，比如下面这个例子：

```python
def get_data(name: str,age: int):
    return name.title(),age # 能够提供 str相关的方法提示
```

你可以声明的数据类型远远不止例子中提到的，所有标准的 `Python` 类型都可声明。

例如：

- `bool`
- `float`
- `bytes`

### 泛型

针对 `python` 中的数据容器，为了声明容器内部元素的数据类型，`python` 提供了对应的类型提示语法（比如：`list[str]`）。

而容器本身也是一种数据类型，其声明了对应变量的数据类型。

像这种能够声明内部元素的类型的类型称为“泛型” 或 “泛型类型”。

下面介绍 `list`、`tuple` 、`set`、`dict` 的对应语法。

#### 列表和集合

下面定义一个由 `int` 组成的 `list` 变量：

```python
def print_list(items: list[int]):
    for item in items:
    	print(item)
```

方括号内的内部类型称为“类型参数”。在这个例子中，`int` 是传给 `list` 的类型，`list` 是传给 `items` 的类型。

`set` 的声明方式与 `list` 完全一样。

#### 元组

为简化代码，后续声明选择性地使用函数。

声明元组的方式与上面有所不同，主要有两种不同的写法：

1. 固定长度，每个位置类型固定

```python
a: tuple[int,str] # 必须是(1,"m")
```

2. 不定长度，类型相同

```python
a: tuple[int,...] 
```

这种写法，长度不固定，但是 `tuple` 内元素均提示为 `int`。

#### 字典

定义 `dict` 时，需要传入两个类型参数，分别针对 `key` 与 `value`。如：

```python
p: dict[str,float]
```

表明：`p` 的键为 `str` 类型；值为 `float` 类型。

#### 联合类型

你可以声明一个变量的类型为若干类型中的一种，这种类型称为“联合类型”。

声明时各类型之间用 `|` 连接即可。如：

```python
item: int | str
```

表明 `item` 可为 `int` 或 `str` 类型。

同时，可以接受 `None` 值：

```python
item: int | None = None # 此处的 = 是赋初值
```

### 类作为类型

在 `Python` 中除了上面提到的基本类型外，也可以把类声明为变量的类型。如：

```python
class Person:
	def __init__(self,name: str):
		self.name = name

def get_name(person: Person):
	return person.name
```

**需要注意的是**：所声明的这个变量是对应类的一个实例，而非类本身。

### typing模块

虽然 `Python3.10+` 能够原生支持类型提示语法，但是为了实现部分特殊要求，可以引入 `python` 标准库中的 `typing` 模块来实现，比如 `Any` 类型提示和元数据注解方法 `Annotated`。

`Any` 类型提示示例：

```python
from typing import Any

a: Any
```

`Annotated` 语法示例：

```python
from typing import Annotated

def process_item(item: Annotated[str, "这是一个字符串"]):  
    print(item)
```

所谓元数据就相当于解释信息，用于说明变量的意义。
