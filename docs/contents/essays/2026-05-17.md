# Teach Yourself Programming in Ten Years 读后感

原文章链接： [Teach Yourself Programming in Ten Years](http://norvig.com/21-days.html)

大多数人都在企求快速地学习编程，很多教程也是以**快速**为噱头。

我之前很长一段时间也是被这困住了，想要学很多东西，但沉不下心去学习，主要是心气高，想要一口吃成个大胖子，企图追上那些沉淀了 10 年之久的大佬（才接触 CS 不到一年的小卡拉米🫨，就想做出优秀的个人项目嘛--简直痴心妄想😂）

这一现象，意味着编程很容易学会吗？并非如此，只有亲身接触了，才会发现，里面的门道太多了😖。

就像 Felleisen 等人编著的书籍《*How to Design Programs*》中提到的那样“Bad programming is easy.Idiots can learn it in 21 days,even if they are dummies.”（中：糟糕的编程很容易，即使是傻瓜，也能在 21 天内学会）

作为名副其实的傻瓜，上述可是由我亲自验证的呢😄

但是，这无法感受到一门语言的魅力，无法理解内在的原理，也就无法熟练地写出优质的代码（就算你喜欢自己的糟糕代码，也要考虑性能和别人-- 看你代码的人 的感受吧😂）。

下面是一个具体的场景，来具体感受这一点。

当前，假设你要通过《24 小时自学 C++》这本书来快速自学 C++。

不妨思考一下，你在 24 小时内，能做什么呢？

很显然，你没多少时间写几个有价值的程序（顶多入门），没时间回顾自己的学习并总结。没有时间学太多的东西，无实践经验，无法深入到 C++的世界中去。

苦学 24 小时，归来仍是门外汉😹

正如 Alan Perlis 所提到的：“A language that doesn't affect the way you think about programming, is not worth knowing”（中：一种不会让你思考编程方式的语言，不值得去了解）。

如果只是在工作过程中，用到了某个内容，不得不快速地学习应用并完成相应任务的话（比如，在写 python 时，你需要用到 C++来优化性能），这是不算学习编程的，而是在学习如何完成当前指定的任务（一旦环境变化，自己就无从下手了）。但是，这并不是在否认这种学习方式（这种学习方式，只能适用在需快速解决某个难题，或者自己的主要方向并不在此）。

研究人员（Bloom （1985）、 Bryan 和 Harter （1899）、 Hayes （1989）、 Simmon 和 Chase （1973））已经证明了这一点，在各种领域积累专业知识大约需要十年时间；Malcolm Gladwell 也提出了“一万小时定律”。

综合以上这些内容，想要真正成为一名专业的程序员，原文总结了一些秘诀：

- 肯花时间（别功利化）
- 兴趣，有兴趣才能肯花时间
- 通过实践来学习，在做中学习
- 交流：多读大佬博客，多看开源项目，多读源码，多去复现（⭐）
- 多专注于**自学**。

 正如《The New Hacker's Dictionary》这本书的作者 Eric Raymond 所说：“Computer science education cannot make anybody an expert programmer any more than studying brushes and pigment can make somebody an expert painter”（中：计算机科学教育不能让任何人成为程序员专家，就像研究画笔和颜料不能让人成为画家专家一样。）

因此，大学教育也不是必需的，就算你只有高中学历，也可能成为技术大牛👍，最重要的是**意识**、**行动**、**总结**（网上优质资源太多了，每个人只要有意识去学去做，都能够取的一定的成果）。

编程语言的学习要尽可能的多（这并不意味着要你都去囫囵吞枣），去理解每一门语言的优缺点，底层实现逻辑，及其特性，学会针对不同的场景采取不同的语言。

原文提到：Remember that there is a "**computer**" in "computer science"。

这意味着，我们无法避开计算机原理（底层）。当然，最推荐的学习方式依旧是**自顶向下、实践驱动**，我们可以从应用出发，但是一定要反哺底层的学习，其可以帮助我们开发更优质的程序，更深入地理解自己开发过程中写的 API 到底在干什么。

总结：从应用出发，反哺底层学习（针对性）；系统学习底层，进而服务于开发。

不要被困于技术的堆砌，多去实践，不是只有学了才能做，是做了才去学。

总而言之，愿意投入大量时间，将生命的一部分用于深思熟虑的实践，不说多牛，至少能取的满意的成就。
